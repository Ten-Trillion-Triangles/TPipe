# P2P Requests and Templates - The Cargo Units

Inter-agent communication in TPipe relies on two specialized request types: the **AgentRequest** (the simplified manual used by LLMs) and the **P2PRequest** (the full industrial cargo unit used by the runtime). **Request Templates** allow you to merge these efficiently, ensuring that every request is properly plumbed with the necessary authentication and security protocols.

Think of a template as a pre-configured cargo container that you fill with specific instructions.

## 1. AgentRequest: The LLM Interface

The `AgentRequest` is a simplified structure designed to be easily generated by an AI model. It contains only the essential valves needed to initiate a call.

```kotlin
val agentCall = AgentRequest(
    agentName = "data-processor",
    prompt = "Analyze this CSV for anomalies",
    content = rawCsvData,
    pcpRequest = optionalToolCall // Optional: Model can request tools via the target agent
)
```

---

## 2. P2PRequest: The Full Cargo Unit

The `P2PRequest` is the complete packet that travels through the switching station. It includes the routing addresses, security credentials, and specialized cargo like context windows and schema overrides.

```kotlin
val cargo = P2PRequest(
    transport = P2PTransport(Transport.Tpipe, "data-processor"),
    authBody = "vault-token-xyz",
    context = myReservoir, // Inject a memory reservoir into the target
    inputSchema = CustomJsonSchema.newSchema(...), // Reshape the target's input
    outputSchema = CustomJsonSchema.newSchema(...) // Reshape the target's output
)
```

---

## 3. Request Templates: The Standard Shipping Profile

Building a full `P2PRequest` manually for every call is inefficient. TPipe uses **Templates** to define a standard shipping profile for an agent.

### Creating a Template
A template stores the static infrastructure requirements for an agent, such as its standard system instructions or authentication tokens.

```kotlin
val processorTemplate = P2PRequest().apply {
    authBody = "standard-access-token"
    prompt.addText("SYSTEM: Always redact PII before returning data.")
    context = sharedReservoir
}
```

### Applying a Template
When an agent (like a **Manifold Manager**) wants to call a worker, it merges the LLM's `AgentRequest` with the pre-configured `P2PRequest` template.

```kotlin
// Build the full request by merging the LLM input with the infrastructure template
val fullRequest = agentCall.buildP2PRequest(processorTemplate)

// Or, use a template already registered in the switching station
val registryRequest = agentCall.buildRequestFromRegistry("data-processor-template")
```

---

## Custom Schemas: Reshaping the Flow

One of the most powerful features of P2P templates is the ability to provide **Custom Schemas**. This allows a caller to reshape the target agent's pipes on-the-fly.

```kotlin
// Define a specialized output format for this specific request
val specializedBlueprint = CustomJsonSchema.newSchema(
    pipeName = "SummaryValve",
    description = "A condensed report format.",
    jsonObject = CondensedReport(id = "", alert = false)
)

cargo.outputSchema = specializedBlueprint
```

When this request is processed, TPipe will **Duplicate** the target agent, fitting its pipes with these new blueprints before execution.

## Key Operational Behaviors

### 1. Template-Driven Security
Templates allow you to hard-code authentication and safety boundaries at the infrastructure level. This ensures that even if an LLM generates a malformed request, the system will apply the correct security cargo (like auth tokens) before the request enters the network.

### 2. Context Injection
A `P2PRequest` can carry an entire `ContextWindow`. This acts as a memory handoff, where one agent can pump its current discoveries into another agent's reservoir, allowing for highly coordinated, multi-agent reasoning.

### 3. Registry Management
Templates can be stored in the `P2PRegistry`, making them globally available to any agent in the yard. This creates a standardized catalogue of services for your infrastructure.
