# MultimodalContent Class API

The `MultimodalContent` class is the "water" flowing through the pipes. It represents the data being processed, containing text, images, documents, and the control signals that tell the pipeline how to behave.

```kotlin
@Serializable
class MultimodalContent(
    var text: String = "",
    var binaryContent: MutableList<BinaryContent> = mutableListOf(),
    var terminatePipeline: Boolean = false,
    var context: ContextWindow = ContextWindow(),
    var miniBankContext: MiniBank = MiniBank(),
    var tools: PcPRequest = PcPRequest(),
    @Transient var modelReasoning: String = "",
    var useSnapshot: Boolean = false
)
```

## Public Properties

### Core Content

**`text`**
```kotlin
var text: String = ""
```
The primary text payload. This is what's usually sent to the model's user prompt.

**`binaryContent`**
```kotlin
var binaryContent: MutableList<BinaryContent> = mutableListOf()
```
A list of binary assets (images, PDFs, etc.) that the model can "see."

**`modelReasoning`**
```kotlin
@Transient var modelReasoning: String = ""
```
The "thought process" generated by reasoning models. This is kept separate from the final text output.

---

### Pipeline Control: Flow Signals

These flags allow pipes to communicate with the parent pipeline, directing the flow of execution.

**`terminatePipeline`**
```kotlin
var terminatePipeline: Boolean = false
```
The "Emergency Stop." If set to `true`, the pipeline stops immediately. Usually set when validation fails critically.

**`repeatPipe`**
```kotlin
@Transient var repeatPipe: Boolean = false
```
The "Recirculation Valve." If `true`, the current pipe will execute again with the same content (useful for multi-step reasoning).

**`passPipeline`**
```kotlin
@Transient var passPipeline: Boolean = false
```
The "Bypass." Allows the pipeline to exit early successfully without running the remaining pipes.

---

### Context Management

**`context`**
```kotlin
var context: ContextWindow = ContextWindow()
```
The local memory reservoir attached to this specific piece of content.

**`miniBankContext`**
```kotlin
var miniBankContext: MiniBank = MiniBank()
```
The multi-page memory reservoir.

**`tools`**
```kotlin
var tools: PcPRequest = PcPRequest()
```
Any tool calls (PCP) generated by the model are stored here before execution.

---

### Metadata & State

**`useSnapshot`**
```kotlin
var useSnapshot: Boolean = false
```
If `true`, TPipe creates a "save state" before sending data to the model. This allows for restoration if a pipe fails and needs to be retried.

**`pipeError`**
```kotlin
@Transient var pipeError: PipeError? = null
```
Detailed error information if a pipe in the mainline "bursts" (fails).

**`metadata`**
```kotlin
@Transient var metadata: MutableMap<Any, Any> = mutableMapOf()
```
A workspace for developers to store arbitrary data that needs to travel with the content through the pipeline.

## Public Functions

### Content Management

#### `addText(additionalText: String)`
Appends text to the current content, automatically adding double newlines for separation.

#### `addBinary(content: ByteArray, mimeType: String, filename: String? = null)`
Adds raw bytes as binary content (e.g., an image or a document).

#### `hasError(): Boolean`
Checks if this content contains error information from a failed pipe.

#### `merge(other: MultimodalContent)`
Merges another content object into this one. This combines text, binary data, memory reservoirs, and metadata.

---

### Pipeline Navigation: Routing the Flow

#### `skipToNextPipe()`
Signals the pipeline to skip the *next* pipe in the sequence and move to the one after it.

#### `jumpToPipe(pipeName: String)`
The "Router." Signals the pipeline to jump directly to a specific named pipe, even if it's earlier in the mainline.

---

### Content Queries

#### `isEmpty(): Boolean`
Returns `true` if there is no text AND no binary content.

#### `shouldTerminate(): Boolean`
Returns `true` if the `terminatePipeline` signal is set or the content is completely empty.

#### `getImages(): List<BinaryContent>`
Filters the binary content to return only images.

#### `getDocuments(): List<BinaryContent>`
Filters the binary content to return only documents (PDFs, text files, etc.).

---

### Utilities

#### `copyMultimodal(): MultimodalContent?`
Creates a deep, serializable copy of the entire content object. Useful for creating snapshots.
