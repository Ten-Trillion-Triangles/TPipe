## P2PDescriptor
Defines an agent's identity and capabilities.

| Property | Type | Description |
|----------|------|-------------|
| `agentName` | String | Unique identifier. |
| `agentDescription` | String | LLM-facing purpose statement. |
| `transport` | P2PTransport | Routing details (local/remote). |
| `requiresAuth` | Boolean | True if transportAuthBody is required. |
| `usesConverse` | Boolean | Uses standard Converse JSON protocol. |
| `allowsAgentDuplication`| Boolean | Permission to clone the pipeline. |
| `allowsCustomContext` | Boolean | Permission to inject raw context. |
| `contextProtocol` | ContextProtocol | tool protocol (pcp, mcp, none). |
| `contextWindowSize` | Int | Token limit for the agent. |
| `supportedContentTypes`| List | Supported MIME types. |

## P2PRequest
Payload for agent-to-agent communication.

- `transport`: Target agent's address.
- `returnAddress`: Where the response should be sent.
- `prompt`: `MultimodalContent` (text, images, etc.).
- `context`: Optional `ContextWindow` injection.
- `authBody`: Credentials for `globalAuthMechanism`.
- `pcpRequest`: tool call details if using PCP.

---

## P2PResponse
Result of a P2P collaboration.

- `output`: `MultimodalContent` on success.
- `rejection`: `P2PRejection` on failure.
  - `errorType`: `auth`, `prompt`, `json`, `content`, `transport`.
  - `reason`: Explanation of the failure.

---

## P2PRequirements
Validation rules enforced by the receiver.

- `requireConverseInput`: Validates JSON structure.
- `allowExternalConnections`: Permits cross-process calls.
- `maxTokens`: Enforces token budget.
- `authMechanism`: Custom lambda for `authBody` validation.

---

## P2PRegistry
Central agent directory.

- `globalAuthMechanism`: A `(suspend (authBody: String) -> Boolean)?` for transport-level security.
- `register(agent: P2PInterface)`: Adds an agent to the system.
- `sendP2pRequest(request: AgentRequest)`: Higher-level helper for LLM tool calls.

---

## P2PHost
Standalone servers for P2P traffic.

### `P2PStdioHost`
- `runOnce()`: Processes one JSON request from `stdin`.
- `runLoop()`: Listens for requests until "exit".

### HTTP Hosting
Accessible via `POST /p2p` when starting TPipe with `--http`.

---

## Supporting Classes

### CustomJsonSchema
Map of `pipeName` to JSON Schema pairs for structured I/O validation.

### AgentRequest
Simplified request generated by models, containing `agentName`, `prompt`, and `content`.

## Enums

- **ContextProtocol**: `pcp`, `mcp`, `provider`, `none`.
- **SupportedContentTypes**: `text`, `image`, `video`, `audio`, `application`, `other`.
- **InputSchema**: `plainText`, `json`, `xml`, `html`, `csv`, `yaml`, `markdown`.
