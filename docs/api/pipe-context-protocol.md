## Overview

The PipeContextProtocol (PCP) package provides a comprehensive tool execution system that allows LLMs to interact with external systems through multiple transport mechanisms including stdio, HTTP, Python, and native TPipe functions.

## Core Classes

### PcpContext

Main configuration container for PCP tool availability and security settings.

```kotlin
@Serializable
data class PcpContext(@Transient val cinit: Boolean = false)
```

#### Public Properties
- `transport: Transport` - Default transport mechanism (Auto, Stdio, Tpipe, Http, Python).
- `stdioOptions`, `tpipeOptions`, `httpOptions`: Lists of available tools.
- `pythonOptions`: Python execution sandbox configuration.
- `allowedDirectoryPaths`, `forbiddenFiles`: Filesystem access control lists.
- `enableSessionAccessControl`, `enableBufferAccessControl`: Multi-user security flags.

---

### PcPRequest

Request object generated by LLMs for tool execution.

```kotlin
@Serializable
data class PcPRequest(
    val stdioContextOptions: StdioContextOptions = StdioContextOptions(),
    val tPipeContextOptions: TPipeContextOptions = TPipeContextOptions(),
    val httpContextOptions: HttpContextOptions = HttpContextOptions(),
    val pythonContextOptions: PythonContext = PythonContext(),
    val argumentsOrFunctionParams: List<String> = mutableListOf()
)
```

---

### PcpExecutionDispatcher

Central dispatcher that routes PCP requests to appropriate executors.

#### Public Functions
- `executeRequest(request: PcPRequest, context: PcpContext)`: Executes a single request.
- `executeRequests(requests: List<PcPRequest>, context: PcpContext)`: Executes multiple requests in parallel using coroutines.

---

### PcpResponseParser

Parses LLM responses to extract and validate PCP requests.

#### Public Functions
- `extractPcpRequests(llmResponse: String)`: Uses robust JSON extraction to find tool calls.
- `validatePcpRequest(request: PcPRequest)`: Checks required fields for the chosen transport.

---

## Context Options

### StdioContextOptions
Configuration for stdio command execution. Includes `command`, `args`, `executionMode` (ONE_SHOT, INTERACTIVE, CONNECT, BUFFER_REPLAY), and `permissions`.

### HttpContextOptions
Configuration for HTTP access. Includes `baseUrl`, `endpoint`, `method`, `allowedHosts` (for SSRF protection), and `authType` (BASIC, BEARER, APIKEY).

### PythonContext
Configuration for Python sandbox. Includes `availablePackages`, `pythonPath`, and `permissions`.

---

## Security Managers

### HttpSecurityManager
Provides SSRF and DNS-rebinding protection. Resolves hostnames once and blocks private IP ranges (127.x.x.x, 192.168.x.x) by default. Enforces `HttpSecurityLevel`.

### PythonSecurityManager
Provides AST-based script validation. Uses `ast_validator.py` to block dangerous code (e.g., `os` imports, `eval` calls) before execution. Enforces `PythonSecurityLevel`.

---

## Standalone Hosting

### `PcpStdioHost` (Object)
Handles tool requests over standard streams.
- `runOnce()`: Processes one `PcPRequest` and exits.
- `runLoop()`: Processes requests in a loop until "exit".

### HTTP Hosting
Accessible via `POST /pcp` when the host is running in HTTP mode.

---

## Result Classes

### PcpRequestResult
Outcome of a single tool call. Contains `success`, `output`, `executionTimeMs`, and `transport`.

### PcpExecutionResult
Aggregate results for multiple concurrent tool calls.

---

## Enums

- **Transport**: `Stdio`, `Tpipe`, `Http`, `Python`, `Unknown`.
- **Permissions**: `Read`, `Write`, `Delete`, `Execute`.
- **StdioExecutionMode**: `ONE_SHOT`, `INTERACTIVE`, `CONNECT`, `BUFFER_REPLAY`.
