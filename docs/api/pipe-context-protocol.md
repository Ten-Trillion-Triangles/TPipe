## Overview

The PipeContextProtocol (PCP) package provides a unified tool execution system that allows LLMs to interact with external systems through multiple transport mechanisms.

## Core Classes

### PcpContext
Main configuration container for PCP tools and security.
- `cinit`: Non-transient flag used for transport identification during deserialization.
- `transport`: Default transport mechanism.
- `stdioOptions`, `tpipeOptions`, `httpOptions`, `pythonOptions`: Available tool definitions.
- `allowedDirectoryPaths`, `forbiddenDirectoryPaths`, `allowedFiles`, `forbiddenFiles`: Filesystem access control.
- `enableSessionAccessControl`, `enableBufferAccessControl`: Multi-user security.

### PcPRequest
Request object generated by LLMs for tool execution.
- `stdioContextOptions`, `tPipeContextOptions`, `httpContextOptions`, `pythonContextOptions`: Populated tool contexts.
- `argumentsOrFunctionParams`: Positional arguments or script content.

### PcpExecutionDispatcher
Central dispatcher that routes PCP requests to executors (Stdio, Http, Python, Tpipe).

## Context Options

All context option classes include a non-transient `cinit` flag to assist the `PcpResponseParser` in identifying the intended transport.

### StdioContextOptions
Configuration for stdio command execution.
- `command`: Binary or script to run.
- `args`: Default arguments.
- `executionMode`: `ONE_SHOT`, `INTERACTIVE`, `CONNECT`, `BUFFER_REPLAY`.
- `permissions`: `Read`, `Write`, `Delete`, `Execute`.
- `timeoutMs`: Execution timeout (default: 30000ms).

### HttpContextOptions
Configuration for HTTP endpoint access.
- `baseUrl`, `endpoint`, `method`: Endpoint definition.
- `allowedHosts`: List of permitted hostnames for SSRF protection.
- `authType`: `BASIC`, `BEARER`, `APIKEY`.
- `permissions`: Required `Read`/`Write` permissions.
- `timeoutMs`: Request timeout (default: 30000ms).

### PythonContext
Configuration for Python script execution.
- `availablePackages`: List of importable packages.
- `pythonPath`: Path to the Python interpreter.
- `workingDirectory`: Execution directory.
- `timeoutMs`: Execution timeout.

## Security Managers

### HttpSecurityManager
Provides SSRF and DNS-rebinding protection.
- Resolves hostnames to IP addresses once during validation.
- Blocks access to private/local network ranges by default.
- Enforces `HttpSecurityLevel` (`STRICT`, `BALANCED`, `PERMISSIVE`, `DISABLED`).

### PythonSecurityManager
Provides AST-based script validation.
- Uses `ast_validator.py` to analyze scripts before execution.
- Blocks dangerous imports (e.g., `os`, `subprocess`) and functions (e.g., `eval`, `open`).
- Enforces `PythonSecurityLevel`.

## Standalone Hosting

### `PcpStdioHost` (Object)
Handles PCP requests over standard streams.
- `runOnce()`: Processes one `PcPRequest` line and exits.
- `runLoop()`: Processes requests in a loop.

### HTTP Hosting
Accessible via `POST /pcp` when the TPipe host is running in HTTP mode.

---

## Result Classes

### PcpRequestResult
Result of executing a single PCP request. Contains `success`, `output`, `executionTimeMs`, `transport`, and optional `error`.

### PcpExecutionResult
Result of executing multiple PCP requests. Aggregates results and overall success status.
