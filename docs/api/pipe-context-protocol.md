## Overview

The PCP package provides a comprehensive tool execution system that allows LLMs to interact with external systems through multiple transport mechanisms.

## Core Classes

### PcpContext
Main configuration container for PCP tools and security.
- **`transport`**: Default transport mechanism (`Auto`, `Stdio`, `Tpipe`, `Http`, `Python`).
- **`stdioOptions`, `tpipeOptions`, `httpOptions`**: Lists of available tool definitions.
- **`pythonOptions`**: Python sandbox configuration.
- **`allowedDirectoryPaths`, `forbiddenFiles`**: Filesystem access control.
- **`enableSessionAccessControl`**: Toggles multi-user isolation.

---

### PcPRequest
Request object generated by LLMs for tool execution.
- **`argumentsOrFunctionParams`**: Positional arguments or script content.
- **`stdioContextOptions`, `httpContextOptions`, etc.**: Per-call configuration overrides.

---

### PcpExecutionDispatcher
Central dispatcher that routes PCP requests to executors.
- **`executeRequest(request, context)`**: Dispatches a single tool call.
- **`executeRequests(requests, context)`**: Executes multiple requests in parallel.

---

### PcpResponseParser
Parses LLM responses to extract tool calls.
- **`extractPcpRequests(llmResponse)`**: Uses robust JSON extraction.
- **`validatePcpRequest(request)`**: Checks required fields for the chosen transport.

---

## Context Options

### StdioContextOptions
Terminal command configuration. Includes `command`, `args`, `executionMode` (ONE_SHOT, INTERACTIVE, CONNECT, BUFFER_REPLAY), and `permissions`.

### HttpContextOptions
Web API configuration. Includes `baseUrl`, `endpoint`, `method`, `allowedHosts` (SSRF protection), and `authType` (BASIC, BEARER, APIKEY).

### PythonContext
Python execution environment. Includes `availablePackages`, `pythonPath`, and `permissions`.

---

## Security Managers

### HttpSecurityManager
Provides SSRF and DNS-rebinding protection. Resolves hostnames once and blocks private IP ranges (127.x.x.x, 192.168.x.x) by default. Enforces `HttpSecurityLevel`.

### PythonSecurityManager
Provides AST-based script validation. Uses `ast_validator.py` to block dangerous code (e.g., `os` imports, `eval` calls) before execution. Enforces `PythonSecurityLevel`.

---

## Standalone Hosting

### `PcpStdioHost` (Object)
Handles tool requests over standard streams.
- **`runOnce()`**: Processes one `PcPRequest` line and exits.
- **`runLoop()`**: Processes requests in a loop until "exit".

### HTTP Hosting
Accessible via `POST /pcp` when the host is running in HTTP mode.

---

## Result Classes

### PcpRequestResult
Outcome of a single tool call. Contains `success`, `output`, `executionTimeMs`, and `transport`.

### PcpExecutionResult
Aggregate results for multiple concurrent tool calls.

---

## Enums

- **Transport**: `Stdio`, `Tpipe`, `Http`, `Python`, `Kotlin`, `JavaScript`.
- **Permissions**: `Read`, `Write`, `Delete`, `Execute`.
- **ParamType**: `String`, `Int`, `Bool`, `Float`, `Enum`, `List`, `Map`.
